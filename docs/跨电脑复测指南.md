# 跨电脑复测指南

本指南用于在一台新电脑上从 GitHub 克隆本项目后，重新运行 Task 3 V2 或 Task 4 V3 的六后端中立评测：

- SenseVoice CER + Whisper CER；
- WavLM SIM + SpeechBrain ECAPA SIM；
- UTMOSv2 + NISQA-TTS。

评测入口只读取已经登记的音频，不会重新调用 TTS 模型。若只需要重新计算分数，不必安装八套 TTS 合成环境；准备 `audio_eval`、评价模型和待复测版本的 24 条克隆 WAV 即可。第 1–12 节以 V2 权威入口为主；V3 的差异和完整命令集中在第 13 节。

## 1. 先理解三层资产

| 层级 | GitHub 克隆后是否存在 | 作用 |
| --- | --- | --- |
| 项目代码、冻结清单、运行记录、3 条参考音频 | 是 | 定义比较对象、目标文本、音频路径和登记哈希 |
| `cloneData/*.wav` 的 24 条克隆音频 | **否**，被 `.gitignore` 忽略 | 六后端的实际被测输入 |
| `audio_eval` 环境和 `hf-mirror` 评价权重 | 否 | 执行 ASR、说话人相似度和自然度预测 |

因此，只执行 `git clone` 不能直接复测。必须从旧电脑或外部制品库复制 24 条克隆 WAV，或者按 [`cloneData/README.md`](../cloneData/README.md) 重新合成；同时需要重新建立评价环境和本地权重目录。

## 2. 已验证基线与资源预算

2026-07-16 完成全量评测的基线如下：

| 项目 | 已验证值 |
| --- | --- |
| 操作系统 | Linux |
| Python | 3.10 |
| GPU | NVIDIA GeForce RTX 4070 Ti SUPER，16 GB |
| NVIDIA 驱动 | 610.47 |
| PyTorch | 2.12.0（运行时显示 CUDA 13.0 构建） |
| torchaudio / torchvision | 2.11.0 / 0.27.0 |
| 评价环境磁盘 | 约 6.7 GB |
| 六后端评价权重与缓存 | 约 4.5 GB |
| 单次 27 音频全量评测 | 串行执行；耗时随 GPU 和首次模型加载状态明显变化 |

建议至少预留 15 GB 可用磁盘。六个后端串行加载，不会同时占用显存；低于 16 GB 的 GPU 可能也能运行，但当前项目没有完成对应硬件验证。CPU 可以通过复制配置并把所有 `device` 改为 `cpu` 运行，但 Whisper、WavLM、ECAPA 和 UTMOSv2 会明显变慢。

## 3. 获取项目代码

```bash
git clone https://github.com/MuYi086/scoring-for-TTS.git
cd scoring-for-TTS
git rev-parse HEAD
```

复测报告中应记录项目提交号。不同电脑比较结果时，先确认提交号、冻结配置和输入音频哈希一致。

## 4. 创建 `audio_eval` 环境

先安装 Miniconda 或兼容的 Conda，再创建只包含 Python 和系统音频工具的基础环境：

```bash
conda env create -f tts-bench/environment/audio-eval-base.yml
conda activate audio_eval
python --version
```

### 4.1 安装 PyTorch

要严格复现 2026-07-16 的已验证组合，可先尝试：

```bash
python -m pip install \
  "torch==2.12.0" \
  "torchaudio==2.11.0" \
  "torchvision==0.27.0"
```

若新电脑的驱动或 GPU 不支持该组合，应按 PyTorch 官方安装选择器安装与本机兼容的 CUDA 版本。此时仍可完成评测，但不应声称与原结果是位级复现；`run_metadata.json` 会记录实际包版本。

验证 GPU：

```bash
nvidia-smi
python -c "import torch; print(torch.__version__, torch.cuda.is_available(), torch.cuda.get_device_name(0))"
```

### 4.2 安装评价依赖

```bash
python -m pip install -r tts-bench/environment/audio-eval-requirements.txt
python -m pip check
```

依赖清单固定了当前已跑通的 FunASR、Transformers、SpeechBrain、NISQA 所需音频栈，并把 UTMOSv2 固定到提交 `cc2700db57bb83ee13dc31ebe1b868c254e15d09`。

## 5. 准备评价模型

统一使用一个仓库外目录。下载阶段不要设置离线变量：

```bash
export HF_MIRROR_ROOT="$HOME/hf-mirror"
export HF_HOME="$HF_MIRROR_ROOT/huggingface-cache"
unset HF_HUB_OFFLINE TRANSFORMERS_OFFLINE
mkdir -p "$HF_MIRROR_ROOT"
```

### 5.1 下载四个直接按目录加载的模型

```bash
hf download FunAudioLLM/SenseVoiceSmall \
  --revision 716d31dbfd64b40152f3399a720890b005ffaf90 \
  --local-dir "$HF_MIRROR_ROOT/FunAudioLLM/SenseVoiceSmall"

hf download openai/whisper-large-v3-turbo \
  --revision 41f01f3fe87f28c78e2fbf8b568835947dd65ed9 \
  --local-dir "$HF_MIRROR_ROOT/openai/whisper-large-v3-turbo"

hf download microsoft/wavlm-base-sv \
  --revision 0a23162ffc49adcf42bdf836a00cb2eb45af3601 \
  --local-dir "$HF_MIRROR_ROOT/microsoft/wavlm-base-sv"

hf download speechbrain/spkrec-ecapa-voxceleb \
  --revision 0f99f2d0ebe89ac095bcc5903c4dd8f72b367286 \
  --local-dir "$HF_MIRROR_ROOT/speechbrain/spkrec-ecapa-voxceleb"
```

### 5.2 下载 UTMOSv2 权重和隐藏依赖

UTMOSv2 的主检查点放在本地目录；其内部还会通过 Transformers 加载 `facebook/wav2vec2-base`，后者必须进入 `HF_HOME` 的标准缓存。

```bash
hf download sarulab-speech/UTMOSv2 fold0_s42_best_model.pth \
  --local-dir "$HF_MIRROR_ROOT/sarulab-speech/UTMOSv2"

hf download facebook/wav2vec2-base \
  --revision 0b5b8e868dd84f03fd87d01f9c4ff0f080fecfe8
```

UTMOSv2 检查点的冻结 SHA-256 为：

```text
c8149d988e4bbf3f347e6966b5d769de347a5f8c59ffca1dc4bd4bf5b8585e57
```

### 5.3 准备 NISQA-TTS

```bash
git clone https://github.com/gabrielmittag/NISQA.git "$HF_MIRROR_ROOT/NISQA"
git -C "$HF_MIRROR_ROOT/NISQA" checkout fe84f0f252abec382b24367d5b22498a7ce34dbb
```

脚本使用 `weights/nisqa_tts.tar`，冻结 SHA-256 为：

```text
c556a954cd9536360b4fe6f524ad432816509cd22f436f345b281a7851360656
```

NISQA 代码使用 MIT License（MIT 许可证）；预训练模型权重使用 CC BY-NC-SA 4.0，商业使用前必须单独核对授权。

完整资产清单见 [`evaluation-assets-v2.json`](../tts-bench/config/evaluation-assets-v2.json)。当前实际占用约为：SenseVoice 901 MB、Whisper 1.6 GB、WavLM 386 MB、ECAPA 86 MB、UTMOSv2 781 MB、Wav2Vec2 缓存 726 MB、NISQA 6 MB。

## 6. 准备 24 条克隆音频

仓库内的 `testData/*.wav` 三条原始参考音频由 Git 管理；`cloneData/*.wav` 被忽略，不会随 GitHub 克隆出现。

### 6.1 推荐：复制既有音频

在旧电脑打包：

```bash
cd /path/to/old/scoring-for-TTS
tar -czf /path/to/cloneData-task3-wav.tar.gz cloneData/*.wav
```

将压缩包传到新电脑后，在新仓库父目录执行：

```bash
tar -xzf cloneData-task3-wav.tar.gz -C /path/to/new/scoring-for-TTS
```

不要把这 24 条音频直接提交到 Git；如需团队共享，应放在外部制品库并保存 SHA-256。

### 6.2 备选：重新合成

若没有旧音频，按 [`cloneData/README.md`](../cloneData/README.md) 和 `modelScript/` 下各模型安装指南建立八套彼此隔离的合成环境，再运行集中克隆脚本。MiMo 使用云端 API，需要 `MIMO_API_KEY`；不得把密钥写入仓库。

重新合成后，既有 `tts-bench/runs/*/synthesis.jsonl` 中的哈希通常会变化。若音频不是原来的字节内容，必须创建新的 `run_id` 和合成记录，不能修改音频后继续冒用旧登记哈希。

## 7. 设置离线运行变量

下载完成后启用离线模式，并把 Matplotlib 与 Numba 缓存放在可写临时目录：

```bash
export HF_MIRROR_ROOT="$HOME/hf-mirror"
export HF_HOME="$HF_MIRROR_ROOT/huggingface-cache"
export HF_HUB_OFFLINE=1
export TRANSFORMERS_OFFLINE=1
export MPLCONFIGDIR="${TMPDIR:-/tmp}/scoring-for-tts-matplotlib"
export NUMBA_CACHE_DIR="${TMPDIR:-/tmp}/scoring-for-tts-numba"
mkdir -p "$MPLCONFIGDIR" "$NUMBA_CACHE_DIR"
```

不要把这些机器专属路径写入项目配置或提交 `.env`。建议把导出命令放在本机 shell 配置或不入库的私有脚本中。

## 8. 一键预检

在真正加载模型前运行：

```bash
conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/check_neutral_evaluation_setup.py \
  --strict-versions
```

预检会验证：

- 已知可用的 Python 与评价包版本；
- CUDA 是否可见；
- 四个本地模型的固定 revision；
- UTMOSv2、NISQA-TTS 文件校验和与 NISQA Git 提交；
- UTMOSv2 所需的 Wav2Vec2 缓存；
- 3 条参考音频、24 条克隆音频、八模型乘三角色完整矩阵；
- 克隆 WAV 当前 SHA-256 是否与 `synthesis.jsonl` 登记值一致。

版本不同但想先做兼容性试跑时，可去掉 `--strict-versions`；缺权重、缺音频、哈希冲突或 CUDA 不可用始终是失败。

## 9. 运行六后端评测

仓库已包含历史结果目录，所以新复测必须指定一个新输出目录，避免覆盖旧证据：

```bash
RUN_ID="replay-$(date -u +%Y%m%dT%H%M%SZ)"
OUTPUT_DIR="tts-bench/reports/$RUN_ID"

conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/run_neutral_evaluation_v2.py \
  --runs-root tts-bench/runs \
  --config tts-bench/config/neutral-evaluation-v2.json \
  --output-dir "$OUTPUT_DIR" \
  --strict
```

评测按后端逐项写入：

- `per_audio.jsonl`：27 条音频的双 CER 与双自然度；
- `speaker_similarity.jsonl`：24 个克隆音频与同角色参考音频的双 SIM；
- `speaker_calibration.jsonl`：3 个同说话人分段对和 3 个跨角色对；
- `run_metadata.json`：配置、包版本和完整覆盖数。

中断后用同一个目录续跑。例如只重跑 Whisper 和 ECAPA：

```bash
conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/run_neutral_evaluation_v2.py \
  --output-dir "$OUTPUT_DIR" \
  --resume \
  --metrics whisper_cer speechbrain_ecapa_sim \
  --strict
```

## 10. 生成三份报告

只有六项覆盖全部完整后，报告生成器才会接受原始结果：

```bash
python tts-bench/scripts/generate_neutral_v2_reports.py \
  --results-dir "$OUTPUT_DIR" \
  --reports-dir "$OUTPUT_DIR/reports"
```

输出名称固定为：

- `SenseVoice_CER&Whisper_CER_V2评价报告.md`；
- `WavLM_SIM&SpeechBrain_ECAPA_SIM_V2评价报告.md`；
- `UTMOSv2&NISQA_V2评价报告.md`。

报告中的证据链接按报告目录与结果目录的相对位置生成，移动整个 `$OUTPUT_DIR` 时仍可保持有效。

## 11. 最终验收

```bash
conda run -n audio_eval python -m pytest -q
python -m pip check
git diff --check
```

然后检查 `$OUTPUT_DIR/run_metadata.json`：六个 `coverage` 项都必须满足 `complete == expected`，逐项错误必须为空。比较两台电脑结果时，至少同时核对：

1. 项目 Git 提交；
2. `neutral-evaluation-v2.json` 和 `evaluation-assets-v2.json`；
3. 27 条输入音频 SHA-256；
4. Python 包版本和模型 revision；
5. GPU、PyTorch 与 CUDA 运行时。

固定种子和五次裁剪控制了 UTMOSv2 的随机输入片段，但不同 GPU、CUDA 或算子版本仍可能造成末位浮点差异。不要只比较格式化报告，应优先比较 JSONL 原始值及容许误差。

## 12. 常见故障

### `输出目录已存在`

新建不同的 `RUN_ID`。只有确认是在修复同一次未完成运行时才使用 `--resume`。

### `HF_MIRROR_ROOT 中找不到评价模型`

目录必须严格匹配模型 ID，例如：

```text
$HF_MIRROR_ROOT/openai/whisper-large-v3-turbo
$HF_MIRROR_ROOT/speechbrain/spkrec-ecapa-voxceleb
```

### UTMOSv2 离线加载 `facebook/wav2vec2-base` 失败

确认下载时已经设置正确的 `HF_HOME`，且下列 revision 存在于标准缓存：

```text
$HF_HOME/hub/models--facebook--wav2vec2-base/snapshots/0b5b8e868dd84f03fd87d01f9c4ff0f080fecfe8
```

### NISQA 报 Matplotlib 或 Numba 缓存不可写

重新设置并创建 `MPLCONFIGDIR`、`NUMBA_CACHE_DIR`，不要依赖只读的用户配置目录。

### SpeechBrain 尝试联网

先运行预检，确认 ECAPA 模型目录完整并保留 Hugging Face revision 元数据；正式运行保持 `HF_HUB_OFFLINE=1`。

### `torch.cuda.is_available()` 为 `False`

先确认 `nvidia-smi` 正常，再确认安装的是 CUDA 版 PyTorch。容器或沙箱还需要显式暴露 GPU；宿主机可见并不代表当前进程可见。

## 13. Task 4 V3 复测

V3 与 V2 共用第 4–5 节定义的 `audio_eval` 环境、六后端权重和 `evaluation-assets-v2.json`。它不复用 V2 的音频或运行记录：

| V3 资产 | 路径 |
| --- | --- |
| 三条原始参考音频 | `testData/mimo_<旁白、小公主、辰南>_v3.wav` |
| 24 条克隆音频 | `cloneData/audio_v3/*.wav` |
| 冻结清单 | `tts-bench/manifests/task4-2026-07-19-v3.jsonl` |
| 合成哈希记录 | `tts-bench/runs-v3/*/synthesis.jsonl` |
| 六后端配置 | `tts-bench/config/neutral-evaluation-v3.json` |

### 13.1 恢复或重新生成 V3 音频

从旧电脑打包：

```bash
cd /path/to/old/scoring-for-TTS
tar -czf /path/to/cloneData-task4-v3-wav.tar.gz cloneData/audio_v3/*.wav
```

在新电脑恢复：

```bash
tar -xzf cloneData-task4-v3-wav.tar.gz -C /path/to/new/scoring-for-TTS
```

若需重新合成，按 [`cloneData/README.md`](../cloneData/README.md) 使用各目录的 `run_clone_*_v3.sh`；MiMo 执行 `python cloneData/mimo/test_clone_mimo_v3.py`。音频字节发生变化后，必须以新 `run_id` 登记新 SHA-256，不得继续冒用仓库内的原记录。大音频不提交到 Git。

### 13.2 V3 严格预检

使用第 7 节的离线环境变量，然后执行：

```bash
conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/check_neutral_evaluation_setup.py \
  --runs-root tts-bench/runs-v3 \
  --config tts-bench/config/neutral-evaluation-v3.json \
  --assets tts-bench/config/evaluation-assets-v2.json \
  --strict-versions
```

预检必须报告 3 条参考音频、24 条克隆音频，并且失败数为 0。

### 13.3 运行 V3 六后端

仓库包含历史 V3 结果；每次复测必须指定新目录：

```bash
RUN_ID="replay-v3-$(date -u +%Y%m%dT%H%M%SZ)"
OUTPUT_DIR="tts-bench/reports/$RUN_ID"

conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/run_neutral_evaluation_v3.py \
  --runs-root tts-bench/runs-v3 \
  --config tts-bench/config/neutral-evaluation-v3.json \
  --output-dir "$OUTPUT_DIR" \
  --strict
```

只有继续这一次未完成运行时才能对相同目录增加 `--resume`。例如只续跑 Whisper 和 ECAPA：

```bash
conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/run_neutral_evaluation_v3.py \
  --output-dir "$OUTPUT_DIR" \
  --resume \
  --metrics whisper_cer speechbrain_ecapa_sim \
  --strict
```

### 13.4 生成 V3 报告

```bash
python tts-bench/scripts/generate_neutral_v3_reports.py \
  --results-dir "$OUTPUT_DIR" \
  --reports-dir "$OUTPUT_DIR/reports"
```

输出名称固定为：

- `SenseVoice_CER&Whisper_CER_V3评价报告.md`；
- `WavLM_SIM&SpeechBrain_ECAPA_SIM_V3评价报告.md`；
- `UTMOSv2&NISQA_V3评价报告.md`。

最终检查 `run_metadata.json` 的六个 `coverage` 项全部满足 `complete == expected`，三份 JSONL 中的 `errors` 全部为空，并按第 11 节运行全量 pytest、`pip check` 与 `git diff --check`。
