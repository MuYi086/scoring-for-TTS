# UTMOSv2 安装与使用说明

UTMOSv2 是六后端 V2 评测中的一个自然度预测器。跨电脑安装的权威步骤见 [`../docs/跨电脑复测指南.md`](../docs/跨电脑复测指南.md)；本文件只说明 UTMOSv2 特有的依赖和复现边界。

## 1. 已验证安装

项目在 `audio_eval` Conda 环境中运行 UTMOSv2。基础环境和完整 Python 依赖分别由以下文件冻结：

- [`../tts-bench/environment/audio-eval-base.yml`](../tts-bench/environment/audio-eval-base.yml)；
- [`../tts-bench/environment/audio-eval-requirements.txt`](../tts-bench/environment/audio-eval-requirements.txt)。

UTMOSv2 固定到官方提交：

```text
cc2700db57bb83ee13dc31ebe1b868c254e15d09
```

安装后验证：

```bash
conda run -n audio_eval python -c "import torch, utmosv2; print(torch.__version__, torch.cuda.is_available(), utmosv2.__version__)"
```

2026-07-16 已验证版本为 `utmosv2==1.3.1.dev0`。

## 2. 两份必需模型资产

主检查点必须位于：

```text
$HF_MIRROR_ROOT/sarulab-speech/UTMOSv2/fold0_s42_best_model.pth
```

SHA-256：

```text

```

UTMOSv2 内部还会加载 `facebook/wav2vec2-base`。离线运行前必须把 revision `0b5b8e868dd84f03fd87d01f9c4ff0f080fecfe8` 下载到 `$HF_HOME/hub/` 标准缓存。只准备主检查点仍会在离线推理时失败。

下载命令和全部评价资产清单见 [`evaluation-assets-v2.json`](../tts-bench/config/evaluation-assets-v2.json)。

## 3. V2 随机性控制

UTMOSv2 默认推理会随机截取音频，单次预测可能随随机状态漂移。V2 冻结配置采用：

```json
{
  "inference_seed": 20260716,
  "num_repetitions": 5,
  "remove_silent_section": true
}
```

每条音频重新设置相同种子，并由模型对五次裁剪取平均。不同 GPU、CUDA 和 PyTorch 算子仍可能产生末位浮点差异，因此不要把固定种子解释为跨硬件位级一致。

## 4. 预检与运行

先执行环境预检：

```bash
conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/check_neutral_evaluation_setup.py \
  --strict-versions
```

只排查 UTMOSv2 时，可在已有 V2 输出目录上续跑：

```bash
conda run --no-capture-output -n audio_eval \
  python tts-bench/scripts/run_neutral_evaluation_v2.py \
  --output-dir /path/to/existing-v2-run \
  --resume \
  --metrics utmosv2 \
  --strict
```

只有继续同一次未完成评测时才能使用 `--resume`；新的比较必须创建新的输出目录。
