# UTMOSv2 安装与使用说明

本项目推荐在现有 `audio_eval` conda 环境中运行自动评估。当前机器已具备 Python 3.10、PyTorch、Transformers、torchaudio 与 soundfile；UTMOSv2 额外需要 `timm`，并会在首次加载预训练模型时取得权重。

## 1. 安装前检查

```bash
conda run -n audio_eval python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
```

预期 CUDA 可用。自动评估依次而非同时加载 WavLM、ASR 和 UTMOSv2，因此不需要为三个评价器叠加预留显存。

## 2. 安装 UTMOSv2

在确认允许联网下载依赖与预训练权重后，按官方版本要求安装。建议先在独立终端执行并保留安装日志：

```bash
conda run -n audio_eval python -m pip install "timm>=1.0.7"
conda run -n audio_eval python -m pip install "git+https://github.com/sarulab-speech/UTMOSv2.git"
```

这两条命令不由自动评估脚本执行。升级前先检查现有环境冲突，避免影响 TTS 合成模型的独立环境。

## 3. 首次准备评价模型

先完成 WavLM、Whisper 和 UTMOSv2 的权重缓存，再执行全量评估。首次允许下载时：

```bash
conda run -n audio_eval python tts-bench/scripts/run_automated_evaluation.py \
  --runs-root tts-bench/runs \
  --config tts-bench/config/automated-evaluation.json \
  --allow-model-download
```

后续离线运行时省略 `--allow-model-download`。若 WavLM 或 ASR 权重不在本地缓存，脚本会把错误写入 `per_case.jsonl`，不会用空值给模型排名。

## 4. 校准后再比较

在配置中的 `ranking.normalizers` 填入校准集确认过的区间。示例值只用于打通流程；若没有校准，不应把 `configured_score` 当作绝对质量分。
