# 样本集约定

评测音频是本地大文件，不提交到仓库。将参考音频置于 `tts-bench/datasets/audio/`，并让清单中的 `reference.audio_path` 使用相对仓库根目录的路径。

一个可比较的样本至少包含：参考音频、参考音频的人工核验转写、待合成文本、语言标记以及预先声明的考察维度。参考音频的转写不能用合成结果的 ASR 输出替代。

建议把样本分为：

- `calibration`（校准集）：用于验证各评价器和人工评分的方向是否合理，不参与最终选型；
- `development`（开发集）：允许模型和提示词迭代；
- `holdout`（保留集）：在候选配置冻结后只运行一次，用于最终比较。

样本的实际选择、授权与脱敏由评估负责人负责；不要在清单、结果或日志中写入不必要的个人信息。
