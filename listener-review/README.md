# listener-review：个人试听复核

本目录仅供你个人独立复核停顿、语气和情绪是否符合创作意图。它不参与 `configured_score`（配置化比较分），也不阻塞 WavLM、ASR、UTMOSv2 和音频健康度的一键自动排名。

自然度、目标音色贴合度、可懂度和可感知伪影已有自动量化信号，但个人试听可用于记录这些模型无法定义的表达取舍，例如“句末是否需要收束”或“角色此处是否应激动”。

## 盲听原则

1. 给听评者的播放列表只显示 `blind_id`，不显示模型、运行编号、文件名中的模型名或上一次评分。
2. 同一 `case_id` 的候选音频按听评者随机化顺序播放；参考音频是否可听必须在评分表中预先声明，并对所有候选一视同仁。
3. 使用耳机、安静环境和固定播放链路。先完成校准样本，再开始正式评分。
4. 一位听评者对同一音频只评一次；同一模型的多个样本应分散在列表中，避免连续听到造成身份猜测。
5. 听评者用匿名 `rater_id`，自由文本评论只描述可听到的现象，不记录个人信息。

管理员用 `private-mapping/` 保存 `blind_id` 与真实 `run_id`/`case_id` 的映射；该目录实际内容被 Git 忽略，以免在盲听结束前泄露候选身份。

评分量表、播放列表模板和结构化字段分别见 `rubric.md`、`forms/` 与 [`contracts/listener-review.schema.json`](contracts/listener-review.schema.json)。
