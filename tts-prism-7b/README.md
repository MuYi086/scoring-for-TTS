# tts-prism-7b：12 维可解释语音诊断

TTS-PRISM-7B 是面向普通话 TTS 的细粒度感知诊断模型。它将“音质好吗”拆分成可定位的维度，并给出文字理由；在本工作区中，它用于解释候选模型在哪类声学或表达维度失分，而不是充当唯一的总裁判。

官方模型卡定义的维度分两层，量纲不能混为一个平均分：

| 层级 | 分数范围 | 维度 |
| --- | --- | --- |
| 基础能力 | 1–5 | 音频清晰度、发音、语调、停顿、语速、说话人一致性、风格一致性、情绪一致性 |
| 高级表现力 | 0–2 加分 | 重音、延长音、副语言、情绪饱满度 |

因此报告时应分别展示基础能力均值、各维度分布和表现力加分；不要把 1–5 与 0–2 的数字不加说明地相加或排名。

## 目录职责

```text
tts-prism-7b/
├── config/       # 模型、音频 tokenizer 与输入输出位置的待填写配置
├── contracts/    # 12 维诊断的结果契约
├── inputs/       # 输入映射说明；不复制 runs 中的原始音频
├── models/       # 本地权重说明；真实权重被 Git 忽略
└── outputs/      # 逐样本诊断结果与原始模型回答
```

## 使用边界

- 仅对完成合成记录的音频诊断；输入音频路径、哈希和模型版本必须写入结果。
- 以官方仓库当时支持的推理输入格式为准。把 TTS-PRISM 的原始回复保存下来，再将可解析的数值写成契约记录；解析失败应标为失败，不能凭感觉补数。
- 它针对普通话诊断设计。非普通话样本、混合语种样本或超出官方能力边界的文本，应在 `run.yaml` 标记为不适用，而不是与普通话分数合并。
- TTS-PRISM 发现的异常必须回听原始音频确认。它能帮助定位，但不能替代受控人工盲听。

模型下载、官方推理实现与许可证以 [TTS-PRISM 官方仓库](https://github.com/xiaomi-research/tts-prism) 和 [模型卡](https://huggingface.co/xiaomi-research/TTS-PRISM-7B) 为准。本仓库不复制其代码、权重或安装命令。
